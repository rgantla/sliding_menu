How does the Sliding Navigation Menu work?

This Sliding Naviagation Menu works based off of hidden checkboxes and menu sheets. Each of the heading menu items is an input of type checkbox with 2 labels. The first label holds the name of the item while the second one holds the back chevron icon. These checkboxes are made invisible using their hidden attribute. Also, the menu items that are children to this main heading are initially floating to the left of the screen by the amount of width of the side navigation.

When the user clicks on one of the heading menu items, the underlying checkbox is set to true and using the CSS3 transition and transform properties the child menu is brought into the view along with a back icon and close icon. The user can either click on the back icon or the close icon to successfully go back to the initial state.

Disclaimer: This sliding menu is Framework / Library agnostic and it does not have any dependencies. The menu itself however, is pretty responsive. You can use the Overflow CSS property to take care of the extra white spaces around the container if any. For Accessibility reasons, you can omit the second label on the chekboxes.
